
\section{实验}
\label{sec:experiments}

在本节中，我们从质量、效率和灵活性三个方面对\model 进行了实验评估。
在第\ref{ssec:class_conditional_synthesis}节，我们在 ImageNet~\cite{deng2009imagenet} 256$\times$256 和 512$\times$512 上评估了\model 的标准类条件图像生成任务。
在第\ref{ssec:applications}节，我们通过展示\model 在三个图像编辑任务上的性能，包括图像修复、外推和编辑，来展示\model 的多功能性。在第\ref{ssec:ablation}节，我们验证了我们的掩码调度设计的必要性。我们将发布代码和模型以实现可重复的研究。


\subsection{实验设置}
\label{ssec:architecture}

对于每个数据集，我们仅训练单个自编码器、解码器和具有 1024 个 token 的码本，用于在裁剪的 256x256 图像上进行所有实验。图像始终以固定的 16 倍因子压缩，即从 $H\times W$ 到大小为 $h \times w$ 的 token 网格，其中 $h$=$H / 16$ 且 $w$=$W/16$。我们发现这个自编码器与码本可以被重复使用来合成 512$\times$512 图像。

本工作中的所有模型都有相同的配置：24 层、8 个注意力头、768 个嵌入维度和 3072 个隐藏维度。我们的模型使用可学习的位置嵌入~\cite{Vaswani17attention}、LayerNorm~\cite{ba2016layer}和截断正态初始化（标准差=$0.02$）。我们采用以下训练超参数：标签平滑=$0.1$、dropout 率=$0.1$、Adam 优化器~\cite{kingma2014adam}，其中 $\beta_{1}$=$0.9$ 且 $\beta_{2}$=$0.96$。我们使用 RandomResizeAndCrop 进行数据增强。所有模型都在 4x4 TPU 设备上以 256 的批大小进行训练。ImageNet 模型训练 300 个 epochs，而 Places2 模型训练 200 个 epochs。 

\subsection{类条件图像合成}
\label{ssec:class_conditional_synthesis}

我们在 ImageNet 256$\times$256 和 512$\times$512 上评估我们模型的类条件图像合成性能。我们的主要结果总结在表~\ref{tab:maintable}中。

\noindent\textbf{质量。}
在 ImageNet 256$\times$256 上，不使用任何特殊的采样策略，如 beam-search、top-k 或 nucleus 采样启发式~\cite{holtzman2019nucleus}或分类器引导~\cite{Razavi19vqvae2}，我们在 Fréchet Inception Distance（FID）~\cite{FID}（$\bestfid$ 对比 $15.78$）和 Inception Score（IS）（$\bestis$ 对比 $78.3$）两方面都显著超越 VQGAN~\cite{Esser21vqgan}。我们也在附录\ref{sec:supp_diversity_comparison}中报告了使用基于分类器的拒绝采样的结果。

%
我们还训练了一个与\model 使用相同分词器和超参数的 VQGAN 基线，以进一步突出双向和单向 Transformer 之间的差异，我们发现在两种分辨率上，\model 仍然显著优于我们实现的基线。

此外，\model 在两种分辨率上改进了 BigGAN 的 FID，在 512$\times$512 上实现了新的最先进技术，FID 为 $7.32$。

\noindent\textbf{速度。}
我们通过评估每个模型生成样本所需的步数（即前向通过）来评估模型速度。如表~\ref{tab:maintable}所示，\model 在两种分辨率上都是所有非 GAN 模型中需要步数最少的。

为了进一步证实\model 和自回归模型之间的速度差异，我们对\model 和 VQGAN 的解码过程进行了运行时间比较。如图~\ref{fig:speed}所示，\model 显著加速了 VQGAN$30$-$64$ 倍，随着图像分辨率（以及输入 token 长度）的增加，加速比变得更加明显。


\noindent\textbf{多样性。}我们除了样本质量外，还考虑分类准确性得分（CAS）~\cite{Ravuri19CAS}和精确度/召回~\cite{KynkaanniemiKLL19}作为评估样本多样性的两个指标。

CAS 涉及首先仅在候选模型生成的样本上训练 ResNet-50 分类器~\cite{ResNet}，然后在 ImageNet 验证集上测量分类器的分类准确度。
表~\ref{tab:maintable}中的最后两列呈现了 CAS 结果，其中包含在真实 ImageNet 训练数据上训练的分类器的得分作为参考（top-1 和 top-5 准确度分别为 76.6\%和 93.1\%）。对于 256$\times$256 的图像分辨率，我们遵循使用数据增强 RandAugment~\cite{cubuk2019randaugment}的常见做法，并在附录~\ref{sec:supp_diversity_comparison}中报告不进行增强训练的得分。我们发现\model 显著优于之前的工作 VQVAE-2 和 VQGAN，在两种分辨率上的 ImageNet 基准上建立了新的 CAS 最先进技术。


表~\ref{tab:maintable}中的精确度/召回结果表明\model 与 BigGAN 相比实现了更好的覆盖（召回），与 VQVAE-2 和扩散模型等基于似然的模型相比实现了更好的样本质量（精确度）。与我们的基线 VQGAN 相比，我们通过召回改进了多样性，同时略微提高了其精确度。

与 BigGAN 的样本相比，\model 的样本更加多样化，具有更多样的照明、姿态、比例和上下文，如图~\ref{fig:diversity}所示。更多比较可在附录~\ref{sec:supp_diversity_comparison}中获得。

\subsection{图像编辑应用}
\label{ssec:applications}
在本小节中，我们展示\model 在三个图像编辑任务上的直接应用：类条件图像编辑、图像修复和外推。如果我们将任务视为对\model 在迭代解码中使用的初始二进制掩码 $\unknownmask$ 的约束，所有这三个任务几乎都可以微不足道地转换为\model 可以处理的任务，如第~\ref{ssec:decoding}节所讨论的。我们展示了在不对架构或任何特定任务训练进行修改的情况下，\model 能够在所有三个应用上生成非常令人信服的结果。此外，\model 在修复和外推上都获得了与专门模型相当的性能，尽管它不是专门为这两个任务设计的。

\noindent\textbf{类条件图像编辑。}
我们定义了一个新的类条件图像编辑任务来展示\model 的灵活性。在这个任务中，模型根据给定的类重新生成边界框内指定的内容，同时保留上下文，即框外的内容。由于违反预测顺序，这对自回归方法来说是不可行的。

但是，对于\model ，如果我们将边界框区域视为迭代解码算法的初始掩码的输入，这就是一个简单的任务。图~\ref{fig:editing}展示了一些例子结果。更多内容可在附录~\ref{sec:supp_image_editing}中找到。

在这些例子中，我们观察到\model 可以合理地替换所选对象，同时保留或在某种程度上甚至完成背景中的上下文。此外，我们发现\model 似乎能够合成在 ImageNet 训练集中未见过的不自然但可信的组合，例如飞行的猫、汤碗中的猫和花中的猫。这表明\model 偶然学到了用于组成的有用表示，这可能在未来的相关任务中进一步被利用。

\vspace{2mm}
\noindent\textbf{图像修复。}
\label{ssec:inpainting}
图像修复或图像补全是一项基本的图像编辑任务，旨在合成缺失区域中的内容，使补全看起来在视觉上逼真。传统的基于补丁的方法~\cite{Barnes:2009:patchmatch}在纹理区域上效果很好，而深度学习的方法~\cite{yu2019free, yi2020contextual, zhao2021comodgan, saharia2021palette, esser2021imagebart}已被证明能合成需要更好语义一致性的图像。这两种方法都在计算机视觉中得到了广泛的研究。

我们通过对掩码的图像进行分词并将修复掩码解释为迭代解码中的初始掩码，将\model 扩展到这个问题。然后，我们通过根据掩码边界线性混合输出图像与输入来合成输出图像，遵循~\cite{cheng2021inout}。
为了匹配我们的基线的训练，我们在 Places2~\cite{zhou2017places}数据集的 512$\times$512 中心裁剪图像上训练\model 。所有超参数都与在 ImageNet 上训练的\model 模型保持一致。

我们将\model 与常见的基于 GAN 的基线进行比较，包括 DeepFillv2~\cite{yu2019free}和 HiFill~\cite{yi2020contextual}，进行具有中央 50\% $\times$ 50\%掩码的修复，在 Places2 验证集上评估。表~\ref{tab:inpaint_uncrop}总结了定量比较。\model 在 FID 和 IS 上都显著超越了 DeepFill 和 HiFill，同时达到了接近最先进的修复方法 CoModGAN~\cite{zhao2021comodgan}的得分。我们在附录~\ref{sec:supp_inpainting_and_outpainting_comparison_with_gans}中展示了与 CoModGAN 的更多定性比较。


\vspace{2mm}
\noindent\textbf{图像外推。}
\label{ssec:outpainting}
外推或图像外推是一项图像编辑任务，最近获得了更多关注。由于来自周围像素的约束较少，因此预测区域中的不确定性更大，被看作比修复更具挑战性的任务。我们对问题的适应和模型的使用与修复中相同。

我们与常见的基于 GAN 的基线进行比较，包括 Boundless~\cite{teterwak2019boundless}、In\&Out~\cite{cheng2021inout}、InfinityGAN~\cite{lin2021infinitygan}和 CoModGAN~\cite{zhao2021comodgan}，进行向右外推 50\%比例的操作。我们在 InfinityGAN~\cite{lin2021infinitygan}和 In\&Out~\cite{cheng2021inout}的作者慷慨提供的图像集上进行评估。

表~\ref{tab:inpaint_uncrop}总结了定量比较。\model 超越所有基线并实现了最先进的 FID 和 IS。如图~\ref{fig:uncrop_diversity}中的例子所示，\model 也能够在给定相同输入但不同种子的情况下合成多样化的结果。我们观察到\model 特别擅长完成对象和全局结构，并假设这是因为模型通过 Transformer 中的全局注意力学到了有用的表示。

我们在 ImageNet 256$\times$256 上使用默认设置进行消融实验。

\noindent{\textbf{掩码调度。}}
\model 的关键设计是在训练和迭代解码中使用的掩码调度函数。我们比较了在第~\ref{ssec:masking}节讨论的函数，在图~\ref{fig:scheduling}中可视化它们，并在表~\ref{tab:ablation}中总结结果。

我们观察到凹函数通常获得比线性更好的 FID 和 IS，其次是凸函数。虽然余弦和平方相对于其他函数表现相似，但余弦在所有得分上都略微超越平方，使余弦成为我们模型的默认值。

我们假设凹函数表现出色是因为它们 1）通过更困难的情况来挑战训练（即鼓励更大的掩码比例），2）在整个解码过程中适当地优先考虑从少到多的预测。
也就是说，过度优先化似乎也是有代价的，如三次函数比平方差和指数函数比所有其他凹函数差得多所示。

\input{figures/ablation_2}

\noindent{\textbf{迭代次数。}}
我们通过用不同的 $T$ 运行所有候选掩码函数来研究迭代次数（$T$）对我们模型的影响。如图~\ref{fig:scheduling}所示，在相同设置下，更多迭代不一定更好：随着 $T$ 增加，除了在整个过程中表现不佳的对数函数外，所有其他函数都达到了``甜点''，其中模型性能在恶化前达到峰值。随着函数变得不那么凹，``甜点''也会``延迟''。如所示，在实现强 FID 的函数中（即余弦、平方和线性），余弦不仅有最强的总体得分，而且在总共 8 到 12 个迭代处有最早的甜点。
我们假设这样的甜点存在是因为太多迭代可能会阻止模型保留较低信心的预测，这会恶化 token 多样性。我们认为对掩码设计的进一步研究对于未来的工作会很有趣。
